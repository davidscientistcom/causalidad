🛑 Más allá de las Correlaciones

Durante más de un siglo, la estadística formal ha operado bajo un dogma paralizante: "La correlación no implica causalidad". Si bien esto es estrictamente cierto, la estadística no nos proporcionó ninguna herramienta matemática formal para expresar qué sí es la causalidad. Este vacío ha limitado enormemente a campos como la medicina, la economía, las políticas públicas y, por supuesto, el Machine Learning.

El Problema Fundamental: Las redes neuronales profundas actuales son, en esencia, sofisticados ajustadores de curvas. Pueden encontrar correlaciones complejas en espacios de alta dimensionalidad P(y|x), pero fracasan cuando se interviene en el sistema o cuando la distribución de los datos cambia (out-of-distribution). La inferencia causal nos da las herramientas matemáticas para ir más allá.

1. El Paradigma de la Estadística Tradicional

La probabilidad y la estadística clásica están diseñadas para responder preguntas sobre la asociación entre variables. El lenguaje formal se basa en distribuciones conjuntas P(X, Y), distribuciones marginales P(X), y probabilidades condicionales:

P(Y | X) = P(X, Y) / P(X)

¿Qué significa P(Ventas | Oferta)? Significa: "Si observo que hay una oferta, ¿cuál es la probabilidad de que las ventas sean altas?".

Pero un gestor no quiere observar una oferta. Quiere hacer una oferta. La estadística clásica no distingue crear una oferta deliberadamente de simplemente ver que el precio estaba bajo porque el producto caducaba. Ambos se colapsan en P(Y|X).

1.1 ¿Por qué la estadística sola no basta?

Consideremos tres escenarios donde la correlación engaña:

🍦 Ejemplo 1: Helados y Ahogamientos

Los datos muestran una fuerte correlación positiva entre la venta de helados y el número de ahogamientos. ¿Debería un gobierno prohibir la venta de helados para reducir ahogamientos?

Realidad: Existe un confusor oculto: la temperatura. Cuando hace calor, la gente compra más helados Y se baña más en piscinas/playas. La temperatura causa ambos eventos independientemente.

La línea punteada roja muestra la correlación espuria; las flechas sólidas muestran la causalidad real. Temperatura es el confusor.

📊 Ejemplo 2: Zapatos y Dolor de Cabeza

Los datos muestran que las personas que duermen con zapatos puestos tienen más probabilidad de despertarse con dolor de cabeza. ¿Dormir con zapatos causa dolor de cabeza?

Realidad: El confusor es el consumo de alcohol. Las personas que beben mucho a veces se duermen vestidas (con zapatos) Y tienen resaca al día siguiente.

🏙️ Ejemplo 3: Bomberos y Daños

Cuantos más bomberos se envían a un incendio, mayor es el daño del edificio. ¿Los bomberos causan daño?

Realidad: La severidad del incendio es el confusor. Incendios más grandes requieren más bomberos Y causan más daño. Si interviniéramos enviando bomberos aleatoriamente (do), veríamos que más bomberos reducen el daño.

1.2 La trampa de la correlación en números

Veamos esto con datos concretos usando Python:

import numpy as np
import pandas as pd

np.random.seed(42)
n = 1000

# La temperatura es la causa real (confusor)
temperatura = np.random.normal(25, 8, n)

# Helados vendidos dependen de la temperatura
helados = 50 + 3 * temperatura + np.random.normal(0, 10, n)

# Ahogamientos dependen de la temperatura (NO de los helados)
ahogamientos = 2 + 0.15 * temperatura + np.random.normal(0, 2, n)

df = pd.DataFrame({'temperatura': temperatura,
                    'helados': helados,
                    'ahogamientos': ahogamientos})

# Correlación ENGAÑOSA helados ↔ ahogamientos
print("Correlación helados-ahogamientos:",
      df['helados'].corr(df['ahogamientos']).round(3))
# Resultado: ≈ 0.67 → ¡fuerte correlación positiva!

# Pero si controlamos por temperatura (regresión parcial):
from sklearn.linear_model import LinearRegression

# Residuales de helados después de quitar efecto de temperatura
res_helados = helados - LinearRegression().fit(
    temperatura.reshape(-1,1), helados).predict(temperatura.reshape(-1,1))

# Residuales de ahogamientos después de quitar efecto de temperatura
res_ahoga = ahogamientos - LinearRegression().fit(
    temperatura.reshape(-1,1), ahogamientos).predict(temperatura.reshape(-1,1))

print("Correlación PARCIAL (controlando temperatura):",
      np.corrcoef(res_helados, res_ahoga)[0,1].round(3))
# Resultado: ≈ 0.01 → ¡Desaparece! No hay efecto causal
Lección clave: La correlación de 0.67 era completamente espuria. Al controlar por el confusor (temperatura), la correlación entre helados y ahogamientos desaparece. Esto es exactamente lo que la inferencia causal formaliza matemáticamente.

2. Correlación Ciega en Machine Learning

El problema no es solo académico. Modelos de ML en producción han causado daños reales por confiar ciegamente en correlaciones.

🏥 Ejemplo 4: El Algoritmo de Asma y Neumonía

Contexto: En un hospital se entrenó una red neuronal para predecir el riesgo de muerte por neumonía, con el objetivo de admitir automáticamente a los pacientes de alto riesgo y enviar a casa a los de bajo riesgo.

El hallazgo de la IA: El modelo aprendió: "Si el paciente tiene asma, el riesgo de muerte por neumonía disminuye. Por tanto, mándalo a casa."

El Error Causal: Médicamente, el asma empeora severamente la neumonía. Pero en los datos históricos, los médicos ingresaban inmediatamente en cuidados intensivos a los pacientes con asma + neumonía. Gracias a esos cuidados agresivos, su supervivencia era excelente. La IA vio la asociación final (Asma → Supervivencia) pero no el mediador (Intervención Médica Intensiva).

🐄 Ejemplo 5: Vacas y Clasificación de Imágenes

Un clasificador de imágenes entrenado para distinguir vacas de camellos aprendió a detectar... hierba verde. En el dataset de entrenamiento, las vacas aparecían siempre en prados verdes y los camellos en desiertos. Cuando se mostró una vaca en la playa, el modelo la clasificó como camello.

Causalidad: La especie del animal causa su apariencia visual (textura, forma). El fondo (prado vs desierto) es un confusor introducido por el sesgo de selección del dataset.

💊 Ejemplo 6: El sesgo de selección en ensayos clínicos

Un modelo predice que cierto fármaco es eficaz basándose en datos observacionales. Pero los médicos recetaban ese fármaco preferentemente a pacientes jóvenes y sanos (los más fáciles de tratar). Al evaluar con un RCT (ensayo aleatorizado), el fármaco resultó ineficaz.

Lección: La asignación del tratamiento no era aleatoria → confusor (gravedad del paciente) sesgaba la correlación observada.

2.1 ¿Cuándo falla el ML clásico?

El ML basado en correlaciones falla sistemáticamente en estos escenarios:

Escenario Pregunta ¿ML correlacional sirve?
Predicción (mismo entorno) "¿Cuánto venderé mañana?" ✅ Sí
Intervención "¿Qué pasa si cambio el precio?" ❌ No
Cambio de distribución "¿Funciona en otro país?" ❌ No
Contrafactual "¿Qué habría pasado si...?" ❌ No
Fairness / Equidad "¿Mi modelo discrimina?" ❌ No

3. La Revolución Causal de Judea Pearl

Judea Pearl (Premio Turing 2011) propuso formalizar un nuevo lenguaje matemático. La idea central es un operador que represente una acción externa, una "cirugía" sobre el modelo probabilístico:

❌ Estadística Clásica (Observar)

P(Y | X = x)

"Si observo que el paciente toma el fármaco, ¿qué probabilidad tiene de curarse?"

Mezcla el efecto del fármaco con las razones por las que se le recetó (gravedad, edad, etc.)

✅ Inferencia Causal (Intervenir)

P(Y | do(X = x))

"Si fuerzo al paciente a tomar el fármaco (independientemente de todo lo demás), ¿qué probabilidad tiene de curarse?"

Aísla el efecto causal puro del tratamiento.

3.1 El operador do() en detalle

El operador do(X = x) significa: "intervenir en el sistema fijando la variable X al valor x, cortando todas las flechas que llegan a X desde sus causas naturales". Esto simula un experimento controlado aleatorio (RCT) pero usando datos observacionales.

P(Y | X = x) ≠ P(Y | do(X = x)) ← ¡Esta es la desigualdad fundamental!

Solo son iguales cuando no hay confusores (cuando X se asigna aleatoriamente, como en un RCT).

3.2 Ejemplo numérico: Ver vs Hacer

💊 Ejemplo 7: Fármaco con confusor

Supongamos que observamos estos datos en un hospital:

Toma fármacoNo toma fármaco
Se cura81%70%
No se cura19%30%

Observando: P(Cura | Fármaco) = 81% > 70%. ¡Parece que el fármaco funciona!

Pero si estratificamos por gravedad (confusor):

Casos LevesCasos Graves
FármacoNo FármacoFármacoNo Fármaco
Se cura93%95%30%35%
No se cura7%5%70%65%

¡Paradoja de Simpson! En AMBOS subgrupos, el fármaco es PEOR. La razón: los médicos daban el fármaco preferentemente a pacientes leves (87% de los que tomaron fármaco eran leves), inflando artificialmente la tasa de curación global.

Intervención: P(Cura | do(Fármaco)) pondera correctamente por la distribución de gravedad en la población, revelando que el fármaco es perjudicial.

4. Las Tres Capas del Razonamiento Causal

Pearl propone que toda pregunta sobre el mundo puede clasificarse en uno de tres niveles (lo veremos en profundidad en la Sección 02):

Peldaño Pregunta típica Formalismo Ejemplo
1. Asociación "¿Qué observo?" P(Y | X) "Los que compran pañales compran cerveza"
2. Intervención "¿Qué pasa si hago?" P(Y | do(X)) "¿Si pongo pañales junto a cerveza, vendo más?"
3. Contrafactual "¿Qué habría pasado?" P(Y_x | X', Y') "Si NO hubiera puesto la oferta, ¿habría vendido igual?"
Cada peldaño es estrictamente más poderoso que el anterior: No puedes responder preguntas del peldaño 2 con información del peldaño 1, ni del peldaño 3 con información del peldaño 2. Un modelo de ML tradicional vive exclusivamente en el peldaño 1.

5.1 Ejemplo rápido con DoWhy

import dowhy
from dowhy import CausalModel
import pandas as pd
import numpy as np

np.random.seed(0)
n = 1000

# Generar datos con confusor Z
Z = np.random.binomial(1, 0.5, n)  # confusor binario
X = 0.6 * Z + np.random.normal(0, 0.3, n)  # tratamiento
Y = 2.0 * X + 3.0 * Z + np.random.normal(0, 1, n)  # outcome

df = pd.DataFrame({'Z': Z, 'X': X, 'Y': Y})

# Definir modelo causal como DAG
model = CausalModel(
    data=df,
    treatment='X',
    outcome='Y',
    common_causes=['Z'],  # declaramos el confusor
    graph="digraph { Z -> X; Z -> Y; X -> Y; }"
)

# Identificar efecto causal
identified = model.identify_effect()

# Estimar efecto causal con backdoor
estimate = model.estimate_effect(
    identified,
    method_name="backdoor.linear_regression"
)
print(f"Efecto causal estimado: {estimate.value:.2f}")
# ≈ 2.0 (el verdadero efecto que pusimos)

# Refutar el resultado (test de robustez)
refutation = model.refute_estimate(
    identified, estimate,
    method_name="random_common_cause"
)
print(refutation)
Nota: Sin ajustar por Z, una regresión naïve daría un coeficiente de ≈3.5 (inflado por el confusor). DoWhy, usando el criterio backdoor, devuelve correctamente ≈2.0.

6. ¿Por qué necesitas esto?

Si trabajas en cualquiera de estas áreas, la inferencia causal no es opcional:

  • Data Science / ML Engineering: Para entender cuándo tu modelo funciona por las razones correctas y cuándo simplemente memorizó atajos espurios en los datos.
  • Economía / Políticas Públicas: Para evaluar el impacto real de intervenciones (no basta con correlacionar gasto público con resultados).
  • Medicina / Epidemiología: Para estimar efectos de tratamientos a partir de datos observacionales cuando un RCT no es ético o es imposible.
  • IA Explicable (XAI): Para responder "¿por qué el modelo decidió esto?" y "¿qué tendría que haber sido diferente para obtener otro resultado?".
  • Fairness / Equidad algorítmica: Para determinar si un algoritmo discrimina causalmente (no solo correlativamente) por raza, género, etc.
⚠️ Sin causalidad, tu modelo es frágil: Un modelo que solo captura correlaciones funcionará bien en el mismo entorno donde se entrenó, pero puede colapsar catastróficamente cuando se despliegue en un contexto diferente, cuando cambien las políticas, o cuando lo uses para tomar decisiones (intervenciones).
