🔧 ML Sesgado vs DoWhy (Backdoor)

En este laboratorio simularemos el problema fundamental del Científico de Datos moderno. Generaremos unos datos con un factor de confusión fuerte y oculto. Entrenaremos un modelo de Machine Learning estándar que sufrirá sesgo brutal, y luego lo solucionaremos implementando matemáticamente la fórmula de Ajuste por Puerta Trasera (Backdoor Criterion) que vimos en el Peldaño 2.

1. El Escenario Patológico (Confounder)

Supongamos que medimos la relación entre las Horas de Entrenamiento Físico (X) de un atleta y su Velocidad en Pista (Y). Sin embargo, hay un factor genético oculto Edad Biológica (Z).
- La gente muy joven tiende a correr más rápido (Z afecta Y).
- La gente muy joven también tiene más energía para entrenar más horas (Z afecta X).

El *verdadero* efecto causal del entrenamiento en la velocidad es exactamente +2.0 por hora. Vamos a generarlo.

import numpy as np
import pandas as pd
from sklearn.linear_model import LinearRegression

np.random.seed(100)
N = 5000

# Z: Edad Biológica (Genética/Metabolismo invertido, mayor es mejor)
Z = np.random.normal(50, 10, N)

# X: Horas Entrenamiento (afectadas por Z, los jóvenes entrenan más)
X = 0.5 * Z + np.random.normal(0, 2, N)

# Y: Velocidad. Causalidad real de Entrenamiento=2.0. Impacto oculto de Z=1.5
# FÓRMULA DIAGNÓSTICA REAL DEL MUNDO: 
Y = 2.0 * X + 1.5 * Z + np.random.normal(0, 1, N)

df = pd.DataFrame({'Entrenamiento_X': X, 'Genetica_Z': Z, 'Velocidad_Y': Y})

2. El Fracaso del Machine Learning Estándar

El equipo de ML puro no tiene ni idea de causalidad. Descargan Entrenamiento_X y Velocidad_Y, e ignoran Z. Entrenan un modelo para medir el impacto de entrenar una hora más.

# Modelo Naïve (Ignorando Z)
X_features = df[['Entrenamiento_X']]
y_target = df['Velocidad_Y']

modelo_ml = LinearRegression().fit(X_features, y_target)
print("Impacto de 1h Entrenamiento (según Scikit-Learn ingenuo):", modelo_ml.coef_[0])
Impacto de 1h Entrenamiento (según Scikit-Learn ingenuo): 4.14

El modelo concluye que 1 hora de entrenamiento aporta 4.14 puntos de velocidad. ¡Es falso y el doble de la realidad (2.0)! Está combinando el efecto del entrenamiento de la persona con la ventaja genética que "coincidentemente" tienen los que más entrenan. Si vendes un plan de entrenamiento prometiendo 4.14, estás estafando al cliente.

3. La Solución: Backdoor Adjustment Programático

El Científico de Datos Causal dibuja un DAG: X ← Z → Y. Se da cuenta de que Z es una puerta trasera abierta. Emplea la poderosa librería DoWhy (o lo codifica a mano si Z es observable) para condicionar y romper el camino espurio.

# === Backdoor Manual con Scikit-Learn ===
# Lo hacemos añadiendo el confounder al modelo de ajuste
X_ajustado = df[['Entrenamiento_X', 'Genetica_Z']]
modelo_causal = LinearRegression().fit(X_ajustado, y_target)

print("Target ATE (Efecto Causal Real Verdadero): 2.000")
print("Impacto de 1h Entrenamiento (modelo Causal Ajustado):", modelo_causal.coef_[0])
Target ATE (Efecto Causal Real Verdadero): 2.000
Impacto de 1h Entrenamiento (modelo Causal Ajustado): 2.012

Lo hemos logrado. Al condicionar estratigráficamente por Z (la puerta trasera), el coeficiente matemático de Regresión del Entrenamiento vuelve a ser la Causalidad Verdadera. El modelo purga la correlación espuria e identifica correctamente el peso físico de las horas.

Nota Práctica: En el mundo moderno de IA empresarial, librerías como DoWhy o CausalML generalizan este bloque de código. Permiten ingresar *DataFrames* puros con miles de variables y un grafo networkx o archivo .dot. El motor examina el DAG de Pearl automáticamente, localiza todos los caminos Backdoor/Frontdoor y expulsa las fórmulas matemáticas de ajuste correctas antes de invocar a los modelos Predictivos clásicos (como Random Forest o un Perceptrón).
