🐍 Hands-on Causalidad: Simulando Grafo y SCM en Python

Toda la teoría de Pearl cobra sentido cuando **controlas el proceso de generación de datos** (Data Generating Process, DGP). Vamos a usar `numpy` para crear un universo artificial donde somos Dios: dictamos exactamente las ecuaciones estructurales causales.

Objetivo del Laboratorio: Queremos reproducir matemáticamente el peligro de un Collider (Colisionador), el error más común del Data Science. Veremos cómo dos variables totalmente independientes desarrollan una "correlación espuria" en Pandas simplemente porque filtramos (condicionamos) el Dataset por la consecuencia común.

1. Definiendo el Modelo Estructural (SCM) usando NumPy

Recordemos el ejemplo clásico: Talento (X) → Hollywood (Z) ← Belleza (Y).

En el mundo real general, el Talento (X) y la Belleza (Y) son exógenos (nacen del azar genético) y son completamente independientes. NUNCA correlacionan. Sin embargo, convertirse en una estrella de Hollywood (Z) es una consecuencia causativa de ambas cosas. Quien tiene las dos, triunfa.

import numpy as np
import pandas as pd
import statsmodels.api as sm

# 1. Fijar semilla para reproducibilidad divina
np.random.seed(42)
n_samples = 100000

# 2. Generar el Ruido Exógeno (U)
# Asumimos que Talento (X) y Belleza (Y) se distribuyen normalmente a nivel global
talento = np.random.normal(0, 1, size=n_samples)
belleza = np.random.normal(0, 1, size=n_samples)

# 3. Ecuación Estructural Causal para Hollywood (Z)
# Si la suma de ambas características excede un umbral (ruidoso), eres estrella actor
factor_xito = 2.0 * talento + 1.5 * belleza + np.random.normal(0, 0.5, size=n_samples)

# Z será 1 si triunfaste en Hollywood, 0 si no.
hollywood = (factor_xito > 3.0).astype(int)

# Montamos nuestro Dataset
df = pd.DataFrame({'Talento_X': talento, 'Belleza_Y': belleza, 'Actor_Z': hollywood})

2. Demostrando la Independencia Global (Peldaño 1)

El Data Scientist "A" descarga el dataset global (100.000 personas de la calle). Chequea si existe alguna relación lineal o correlativa entre el Talento y la Belleza usando una regresión OLS.

# Regresión global de Belleza predicha por Talento
X_global = sm.add_constant(df['Talento_X'])
model_global = sm.OLS(df['Belleza_Y'], X_global).fit()
print("Coeficiente del Talento a nivel global:", model_global.params['Talento_X'])
Coeficiente del Talento a nivel global: 0.0019 (p-value: 0.542 No Significativo)
Correlación: 0.00 → Es decir, ¡total independencia! (Correcto causalmente)

3. El Error Causal: Condicionar por el Collider Z

El Data Scientist "B" es contratado por unos estudios en Los Ángeles. Solo tiene datos de la industria del cine. Es decir, su `DataFrame` está filtrado implícitamente: df_hollywood = df[df['Actor_Z'] == 1]. ¡Está condicionando por el Collider! 

# Creamos el Dataset condicionado a Hollywood
df_hollywood = df[df['Actor_Z'] == 1]

# Regresión de Belleza predicha por Talento EN HOLLYWOOD
X_holly = sm.add_constant(df_hollywood['Talento_X'])
model_holly = sm.OLS(df_hollywood['Belleza_Y'], X_holly).fit()
print("Coeficiente del Talento en Hollywood:", model_holly.params['Talento_X'])
Coeficiente del Talento en Hollywood: -0.68 (p-value: 0.0000 Extremadamente Significativo)
Correlación: Furiosamente Negativa. La IA concluye: "Mientras más talento tiene una persona de la base de datos, ¡más *fea* es!"

El Descubrimiento de Pearl Empírico: La estadística cruda no se ha equivocado aritméticamente; en ese dataset de actores, el talento y la belleza muestran correlación negativa. El que ha fallado es el Científico de Datos por no entender el DAG. Al filtrar por `Actor_Z` (condicionar por el colisionador), abrió forzosamente un flujo de información (correlación) entre dos variables que eran originalmente independientes.

Esto se conoce como Sesgo de Selección o Berkson's Paradox. Lo único que lo soluciona es entender a nivel de grafo que Z es un colisionador entre X e Y, y por lo tanto, según las leyes de la d-separación de Pearl, nunca se debe intentar introducir Z en el modelo probabilístico si se quiere evaluar la relación pura entre X e Y.
